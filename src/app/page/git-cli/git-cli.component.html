The git CLi

As a developer, I'm using different tools every day.

One of my favorite tools is a git.

There are plenty of ways of using git. As I mentioned in the prerequisites section, there is a set of GUI software.

But I prefer to use git from command-line.

So, here are some of the commands I'm using daily.

init

If you need to initialize a new repository run this command:

git init

That's all! Now you have the initialized repo. Make your first commit.

pull

If you are working as a part of a team, or if you are working from different workstations, you need to keep your local repo up to date.

To perform this run the following:

git pull

After running this command, don't forget to check if there are any merging conflicts.

branch

There can be several branches in a repository.

Next command is showing a list of branches and highlights the active one:

git branch

It is a bad habit to work on the master or develop branch.

Most of the work is happening in child branches.

To create a new branch, run this command:

git branch branch_name

If you want to remove local branch type this command:

git branch -d branch_name

Don't forget that the branch will remain on a remote server.

checkout

To switch to another branch issue the next command:

git checkout branch_name

If you want to create a new branch and switch to it, add -b option:

git checkout -b branch_name
add

After you have finished your work, you need to stage files. To add all files at once use this command:

git add --all

Or you can add files using the following syntax:

git add path_spec Note that path_spec can be a single file name, a path to a directory, and also it supports wildcards.
commit

Once you have staged files, you can commit them to the remote repository. One option is to use the abilities of your IDE. But you can file the next command:

git commit

As a result, the git will open editor which is set in its configuration and will wait until you save the generated file. This option allows you to write commit's comments longer than 70 symbols.

If you prefer one-line comment style then add your description using this syntax:

git commit -m "your_comment_here"
merge

Merging branches can be done as follows:

git merge branch_name

Don't forget to check for conflicts and fix them if they arise.

It is also a good practice to merge branches via pull requests instead of doing this directly.

push

Pushing your changes to a remote repository is done like this:

git push

If your branch exists only on your local workstation, then the git will warn you about this. But if you are aware of that, you can run this command:

git push --set-upstream origin branch_name

This command will create a new branch in a remote repository and push your changes.

help

If you stuck at some point, you can always read the git manual. To ask git for help type:

git command --help
summary

There are a lot of abilities of git which are not covered by this article. But this is the minimal set for everyday usage.
