In this demo, I've tried to simplify the entire process of rendering of the object.

Static parts reside in the CSS definitions. And the dynamic parts are calculated upon each parameters update.

Because this object has a simple structure, all data required to build it is calculated with the help of three functions.

The two of them faceWidth and faceHeight are calculating the correct width and height of each part.

The transform function finds which transformations are required to place each part in the proper place and to turn them to the right angle.

Description

The idea behind the rendering process is simple.

There is an object (I refer to it as plane) that defines the height and width of the object. By default the plane is invisible.

Next, I'm creating other objects (I refer to them as faces). Each face is placed in the correspondent place with the respective angle.

The set of faces (I refer to them as bar) is positioned from top of the plane to the bottom in order of appearance. Thus, by defining the number of bars, it is possible to realign faces to the right places.

In this demo, the only rotating object is the plane. All other parts are just following the plane.

If, for some reason, the browser doesn't support the yet experimental CSS feature transform-style there would be no 3D effect. To try it out, you may toggle an appropriate checkbox in the demo.

Conclusion

With this new feature, it is possible to enrich web pages with simple 3D objects.

I've got a couple of ideas about how it can be applied!

It can be a preloader to your web application.

It is possible to visualize some data.

Also, it is possible to implement some fancy 3D spinners/waiters/loading indicators!
